# Jarvis: 업무용 데스크톱 비서

Jarvis는 Windows 환경에서 업무 효율을 높여주는 개인용 AI 도우미입니다. 사내 규정 확인부터 PC 내 파일 검색, 실시간 정보 확인까지 한곳에서 처리합니다.

## 📌 1. 프로젝트 개요

**Jarvis**는 흩어져 있는 정보(사내 PDF 규정, 로컬 파일, 웹 실시간 데이터)를 하나로 연결해 주는 어시스턴트입니다. 단순히 챗봇과 대화하는 것을 넘어, 실제 업무에 필요한 데이터를 즉시 찾아주는 데 초점을 맞췄습니다.

### 주요 기능
- 🤖 **로컬 LLM 구동**: Ollama를 사용해 인터넷 연결 없이도 안전하게 대화가 가능합니다.
- 📚 **사내 문서 검색 (RAG)**: 복잡한 PDF 규정을 학습해 질문에 맞는 정확한 근거를 찾아 대답합니다.
- 🔍 **PC 파일 초고속 검색**: Everything 엔진을 연동해 컴퓨터 내 수백만 개의 파일을 1초 만에 찾아냅니다.
- ⚡ **실시간 정보 수집**: 날씨, 주가 등 최신 정보를 네이버에서 직접 스크래핑하여 사실에 기반한 데이터만 제공합니다.
- 📈 **주식 분석 서비스**: 삼성전자, 네이버 등 주요 종목의 기술적 지표(RSI, MACD 등)를 `yfinance`로 계산해 분석해 줍니다.
- 🖥️ **쾌적한 UI**: PyQt5 기반의 깔끔한 인터페이스와 끊김 없는 타이핑 효과를 제공합니다.
- 🛡️ **간결한 응답 스타일**: 불필요한 인삿말을 빼고, 요청한 핵심 내용만 즉시 답변하는 비서 모드를 적용했습니다.

### 🛠️ 사용한 기술 (Tech Stack)

**Core & GUI**
- **Python 3.12.2 / PyQt5**: 메인 로직 및 데스크톱 인터페이스 구현
- **QThreadPool**: 무거운 렌더링 작업을 백그라운드에서 처리해 UI 버벅임 방지

**AI & NLP**
- **Ollama (`llama3.1:8b`)**: 로컬 언어 모델 코어
- **ChromaDB**: 문서 검색을 위한 벡터 데이터베이스
- **SentenceTransformers**: 텍스트 문맥 분석 및 실시간 임베딩

**데이터 수집**
- **Everything**: 윈도우 파일 시스템 기반의 고속 탐색 엔진
- **BeautifulSoup4**: 네이버 실시간 데이터 추출
- **yfinance / Tavily API**: 금융 데이터 및 웹 검색 연동

## 🚀 2. 시작하기

### 🥇 일반 사용자용 (실행 파일)
개발 환경 설정이 번거롭다면 빌드된 파일을 바로 사용하세요.

1. **Ollama 설치**: `https://ollama.com`에서 다운로드 후 터미널에서 `ollama pull llama3.1:8b` 실행
2. **Jarvis 실행**: 배포 폴더 내의 `jarvis.exe` 실행

### 🥈 개발자용 (소스 구현)
직접 코드를 수정하거나 기능을 확장하고 싶다면 아래 과정을 따르세요.

```powershell
# 1. 가상환경 및 패키지 설치
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

# 2. 환경 설정
copy .env.example .env  # .env 파일에 필요한 API Key 입력

# 3. 앱 실행
python jarvis.py
```

## 👤 3. 주요 활용 가이드

- **웹 검색**: "세종 일기예보 알려줘", "카카오 주가 어때" 등 실시간 데이터가 필요한 질문에 네이버 정보를 바탕으로 정확히 대답합니다.
- **주식 지표 확인**: "`삼성전자 분석해줘`", "`RSI 알려줘`" 등으로 질문하면 기술적 지표를 수치로 환산해 해석해 줍니다.
- **사내 문서 질문**: "회사 연차 규정 알려줘"처럼 data 폴더에 있는 PDF 내용을 바탕으로 규칙을 찾아줍니다.
- **PC 파일 탐색**: "2024년 기획서 파일 찾아줘"라고 하면 컴퓨터 내의 관련 파일을 리스트로 보여줍니다.

## 💻 4. 개발자 참고 사항

### 저장소 및 폴더 구조
```text
Root/
├── src/                  # 메인 소스 코드
│   ├── app/              # 채팅 창 UI 및 AI 응답 처리 워커
│   ├── core/             # 라우팅 매니저, 경로 관리 등 공통 코드
│   ├── services/         # 검색 엔진(Everything), 웹 스크래핑 모듈
│   ├── repositories/     # 채팅 로그 및 이력 저장소
│   ├── ui/               # 말풍선, 렌더러 등 PyQt5 커스텀 위젯
│   └── rag/              # 문서 분석 및 RAG 전용 스크립트
├── data/                 # 사용자 데이터 (Git 제외)
│   ├── documents/        # 학습용 PDF 원본 보관
│   ├── vectordb/         # 분석된 문서 조각 데이터 (Chroma DB)
│   ├── model_cache/      # 오프라인 구동을 위한 AI 모델 캐시
│   └── chats/            # 대화 이력 저장 폴더
├── assets/               # 아이콘 및 Everything 검색 엔진 본체
├── scripts/              # 앱 실행 및 빌드용 배치 파일
└── packaging/            # 실행 파일(exe) 빌드 설정
```

### 문서 갱신 방법 (RAG)
사내 규정이 바뀌어 PDF를 새로 넣었다면 아래 명령어를 실행해야 합니다.
1. `data/documents` 폴더에 새 PDF 업로드
2. `python src/rag/ingest.py` 실행 (기존 데이터 초기화 및 재색인)

## 📦 5. 최적화 및 배포 관리

### 실행 파일(.exe) 빌드
현재 작동이 확인된 소스코드와 최신 데이터를 모아 단일 실행 파일로 압축합니다.

```powershell
scripts\build_jarvis.bat
```
빌드가 완료되면 `dist\jarvis\jarvis.exe`가 생성되며, 최종 배포를 위한 `jarvis_package.zip` 파일이 루트 폴더에 만들어집니다.

### 빌드 속도 및 용량 팁
- 앱 용량의 대부분(약 450MB)은 오프라인 텍스트 분석을 위한 모델 캐시(`data\model_cache`)입니다. 100% 로컬 구동을 위해 필요한 구성 요소입니다.
- `build` 폴더를 유지하면 재빌드 시간을 단축할 수 있습니다.
- 백신 프로그램이 `dist`나 `build` 폴더를 실시간 검사할 경우 빌드 속도가 느려질 수 있으니 예외 등록을 권장합니다.

## ❓ 6. 자주 묻는 질문 (FAQ)

**Q. 답변이 너무 느리거나 화면이 멈춰요.**
- 저사양 PC를 위해 마크다운 변환과 타이핑 애니메이션을 비동기로 처리하도록 최적화했습니다. 만약 느리다면 Ollama 모델이 실행 중인지, 리소스가 부족하지 않은지 확인해 주세요.

**Q. 사내 문서를 물어봐도 엉뚱한 대답을 합니다.**
- PDF를 수정한 뒤 `ingest.py`를 실행했는지 확인해 주세요. 최신 데이터가 벡터 DB에 반영되지 않았을 가능성이 큽니다.

**Q. 왜 LangChain 같은 프레임워크를 쓰지 않았나요?**
- **빠른 반응 속도**: 에이전트가 의도를 판단하기 위해 LLM을 한 번 더 거치는 시간을 줄여, 첫 글자가 나오는 속도를 높였습니다.
- **빌드 효율성**: 큰 라이브러리 의존성을 줄여 실행 파일 용량을 가볍게 유지하고 빌드 시간을 단축했습니다.
- **상세 제어**: 프롬프트 흐름을 직접 관리하여 답변의 일관성을 높이고 불필요한 설명을 배제했습니다.

## 🤖 7. 개발 뒷이야기 (Vibe Coding)

이 프로젝트는 단순히 코드를 짜는 것을 넘어, AI 에이전트와 대화하며 구조를 설계하고 구현하는 **"Vibe Coding"** 방식으로 제작되었습니다.

- **모델별 역할 분담**: Claude로 아키텍처를 잡고, Codex로 핵심 로직을 짰으며, GPT로 디버깅을 진행했습니다.

### 🧠 협업하며 느낀 모델별 강점
- **Claude**: 전체적인 설계와 큰 틀을 잡는 능력이 탁월합니다. 복잡한 리팩토링이나 구조를 정의할 때 시니어 개발자 같은 조언을 해줍니다.
- **GPT-4/5**: 디버깅과 도구 연동에서 실무적인 해결책을 잘 제시합니다. 에러 수정이나 세부적인 로직 보완에 강점이 있습니다.
- **Codex**: 핵심 실무 로직을 군더더기 없이 구현해내는 속도가 압도적입니다. 논리적인 코드 생성 능력이 매우 뛰어납니다.
- **Gemini**: 방대한 문서 분석이나 외부 레퍼런스 조사가 필요할 때 박학다식한 보조 개발자 역할을 톡톡히 해냅니다.

- **에이전트는 조력자**: AI는 개발 루프를 비약적으로 줄여주지만, 최종적인 컨텍스트 관리와 의사결정은 여전히 개발자의 몫임을 확인한 실험이었습니다.

> **"AI는 가속 페달일 뿐, 핸들은 여전히 사람이 잡아야 합니다."**
